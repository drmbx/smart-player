# 🎬 Smart Player / Умный проигрыватель

Десктопное приложение для интерактивной визуализации и автоматической детекции мешковидных (bag) структур в дисперсно-кольцевых газожидкостных течениях. Построено на базе `PyQt6` и `Ultralytics YOLO26`, поддерживает асинхронную обработку, динамическую фильтрацию по порогу уверенности и экспорт результатов в машиночитаемые форматы.

---

## 🚀 Особенности
- ⚡ **Асинхронный инференс**: вычисления вынесены в `QThreadPool`, интерфейс не блокируется при обработке сотен кадров.
- 🎛 **Интерактивная настройка**: смена весов модели и порога уверенности в реальном времени без перезапуска.
- 🔍 **Умная навигация**: быстрый переход к кадрам, содержащим детекции выше заданного порога.
- 📤 **Экспорт**: сохранение аннотаций в `CSV` и генерация изображений с наложенными рамками (`PNG`).
- 💻 **Кроссплатформенность**: корректная работа на Windows и Linux, автоматическое определение GPU/CPU.

## 🛠 Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/drmbx/smart-player.git
cd smart-player
```

### 2. Установка менеджера пакетов `uv`
`uv` обеспечивает быстрое и детерминированное окружение. Установка одной командой:
```bash
# Windows (PowerShell)
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# Linux / macOS
curl -LsSf https://astral.sh/uv/install.sh | sh
```
*После установки перезапустите терминал или добавьте путь в `PATH`.*

### 3. Создание окружения и установка зависимостей
```bash
# Создание виртуального окружения
uv venv

# Активация (Windows)
.venv\Scripts\activate
# Активация (Linux/macOS)
source .venv/bin/activate

# Установка зависимостей из pyproject.toml
uv sync
```

### 4. Подготовка весов модели
Создайте директорию `weights/` в корне проекта и поместите туда файл весов в формате `.pt`:
```bash
mkdir weights
# Пример: скопируйте обученный файл
cp путь/к/вашей/модели/best.pt weights/
```

### 5. Запуск приложения
```bash
uv run main.py
```
При первом запуске автоматически загрузятся необходимые зависимости `PyTorch`, `Ultralytics` и `PyQt6`.

## 🖥️ Краткий гайд по использованию
1. **Загрузка данных**: нажмите `📁 Папка с изображениями…` и выберите директорию с кадрами (`PNG`, `JPEG`, `BMP`).
2. **Выбор модели**: нажмите `Выбрать веса` и укажите путь к `.pt` файлу.
3. **Детекция**: нажмите `🔍 Детекция`. Прогресс отобразится в интерфейсе.
4. **Фильтрация**: двигайте ползунок `Порог уверенности` для мгновенного скрытия/показа рамок.
5. **Навигация**: используйте `▶/⏸`, `◀ Кадр ▶`, `◀ Объект ▶` или ползунок времени.
6. **Экспорт**: 
   - `📄 Экспорт CSV` → таблица с координатами, классами и confidence
   - `🖼️ Экспорт изображений` → папка с кадрами, на которые наложены рамки

## ⚠️ Примечания и устранение неполадок

| Проблема | Решение |
|----------|---------|
| `ModuleNotFoundError: No module named 'PyQt6'` | Убедитесь, что окружение активировано: `.venv\Scripts\activate` (Win) или `source .venv/bin/activate` (Linux) |
| Медленная обработка на Linux | Проверьте, используется ли GPU: в консоли должно выводиться `[YOLO] Инициализация на устройстве: CUDA`. Если нет, установите `torch` с поддержкой CUDA: `uv pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121` |
| Ошибки GUI на headless Linux | Установите системные библиотеки: `sudo apt install libgl1 libglib2.0-0 libsm6 libxext6 libxrender-dev` |
| Приложение "зависает" при экспорте | Экспорт выполняется асинхронно. Не закрывайте окно до появления диалога `Успех`. |
| Нет детекций после запуска | Убедитесь, что изображения в формате `PNG/JPEG/BMP`, а имя класса в модели совпадает с разметкой (по умолчанию ожидается `bag` или `0`). |
